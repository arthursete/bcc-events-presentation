Existing Solution Analysis

Brisbane City Council Open Data Portal (data.brisbane.qld.gov.au) — Powered by Opendatasoft

Solution Overview

The Brisbane City Council Open Data Portal is a government data publishing platform hosting 384+ datasets for public access. It is not designed as an end-user application for finding events or venues — it is a data catalogue primarily intended for developers, researchers, and data analysts.

The Events Locations dataset contains 1,696 event venue records across 186 suburbs, published under a Creative Commons Attribution 4.0 licence. It can be downloaded in CSV, JSON, Excel, and GeoJSON formats.

How It Works

Users must navigate through the catalogue of 384+ datasets to find the Events Locations dataset, then choose between viewing tabs:

ViewDescription
TablePaginated spreadsheet-style table with raw venue data in columns
MapLeaflet.js interactive map with venue markers; shape-drawing spatial filters
AnalyseBasic chart builder for simple visualisations (cannot save)
Export/APIDownload in CSV, JSON, Excel, GeoJSON; Opendatasoft Explore API

Pros & Cons

Pros (7)

  • Comprehensive raw data access — Full dataset freely available in multiple formats
  • API access — Programmatic access via Opendatasoft Explore API
  • Geographic visualisation — Built-in map view with spatial filtering
  • Multiple related datasets — Cross-reference with Library, Music, Markets events
  • Open licence — CC BY 4.0 allows free reuse
  • Daily updates — Refreshed daily from Trumba Calendar API
  • Multiple export formats — CSV, JSON, Excel, GeoJSON

Cons (10)

  • Not for general public — Data catalogue, not venue discovery tool
  • No suburb browsing — Can't type a suburb and see nearby venues
  • No visual type filtering — Raw field values, no icons or categories
  • No data integration — Locations and events are separate datasets
  • Raw data presentation — Coordinates, field names, unformatted text
  • No personalisation — No favourites, recommendations, or preferences
  • Poor mobile experience — Desktop-focused interface
  • No proximity search — No "near me" or distance radius filtering
  • Overwhelming UI — API docs, JSON schemas, metadata visible
  • Limited events — Trumba cap restricts to 1,000–2,000 items

Key Pages

Explore Catalogue

The main entry shows 384+ datasets as cards. Users must search or browse theme categories to find Events Locations. No prominent "Events" or "Venues" section exists.

Table View

A paginated spreadsheet showing all 1,696 records. Plain, no visual hierarchy, icons, or colour coding. Users must page through or search to find venues.

Map View

Venues plotted as markers on a Leaflet.js map with multiple basemaps. Geographic shape-drawing tools allow spatial queries but are unintuitive for casual users. No suburb boundaries, no type clustering.

Metadata View

Technical metadata: dataset identifier, CC BY 4.0 licence, JSON schema, download statistics. Useful for developers, irrelevant for general users.

Gap Analysis — Why a New Solution is Needed

The BCC Open Data Portal is excellent for raw data access but fundamentally fails as a tool for the general public to discover event locations. Its data-catalogue design, technical interface, lack of cross-dataset integration, absence of suburb-based browsing, and raw data presentation create significant barriers for non-technical users.

This gap represents a clear opportunity for BCC Events Finder — a purpose-built web application that transforms the same data into an accessible, visually engaging venue discovery experience.
