BCC Events Finder: A Web Application for Brisbane City Council Event Venues

Contents 1. Introduction 2. User Story 3. Data Story 4. Proposed Solution

1. Introduction

This technical proposal presents the design of BCC Events Finder, a purpose-built web application that transforms the Brisbane City Council Events Locations dataset into an accessible, user-friendly venue discovery platform. While BCC publishes comprehensive venue data through their Open Data Portal, this information is presented in a raw, technical format inaccessible to everyday residents and visitors.

2. User Story

2.1 Description of Web Application

BCC Events Finder provides Brisbane residents and visitors with an intuitive way to discover, search, and filter the 1,696 event venue locations maintained by Brisbane City Council.

Public User Features

  • Home page with prominent search bar and quick venue type filter buttons
  • Browse/search page with sidebar filters (suburb dropdown, venue type checkboxes), sort options, paginated results
  • Venue detail page with address, type, suburb, map placeholder with GPS coordinates
  • Related venues section showing other venues in the same suburb

Admin Features

  • Secure login with hashed password authentication
  • Dashboard with statistics and searchable venue management table
  • Add/Edit venue form with all data fields
  • Delete venue with confirmation prompt

2.2 Existing Solution

The Brisbane City Council Open Data Portal (data.brisbane.qld.gov.au) is the existing solution. Powered by Opendatasoft, it hosts 384+ datasets including the Events Locations dataset. It provides table view, map view, analyse view, and export/API access.

Key Pros

  1. Comprehensive raw data access in multiple formats (CSV, JSON, Excel, GeoJSON)
  2. API access for developers via Opendatasoft Explore API
  3. Geographic visualisation with interactive Leaflet.js map
  4. Open licence (CC BY 4.0) allows free reuse
  5. Daily updates from Trumba Calendar API

Key Cons

  1. Not designed for general public — it's a data catalogue for developers
  2. No suburb-based browsing or friendly location search
  3. No visual venue type filtering (raw field values only)
  4. No integration between location and event datasets
  5. Raw data presentation with coordinates and field names
  6. No personalisation, favourites, or recommendations
  7. Poor mobile experience — desktop-focused design
  8. No proximity/distance search
  9. Overwhelming technical interface for non-technical users
  10. Limited to 1,000–2,000 events due to API cap
